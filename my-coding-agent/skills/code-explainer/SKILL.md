# 代码解读技能

只读，不改代码。

## 使用场景
- 用户让你解释某段代码的作用
- 用户看不懂某个函数的逻辑
- 用户想了解项目的整体架构

## 输出格式
1. 概述：这个文件/函数是干什么的
2. 输入→处理→输出 流程
3. 依赖关系：调用了谁，被谁调用
4. 值得注意的设计点和潜在坑

## 约束
- 禁止执行 write_file / edit_file
- 生成的文件只能写入 test/ 目录
